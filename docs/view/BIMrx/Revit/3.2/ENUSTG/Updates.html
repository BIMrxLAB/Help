The v3.2.0.2 release of BIMrx® for Revit highlights the following updates:


BIMrx Revit Feature fixes:


  • BIMrx Core Renumber – UI repaired to show the ‘X’ button to delete custom changes made to the order sequence
  • BIMrx Core Find and Replace – Load time optimization when selecting elements with many shared parameters. Load times of the UI should be significantly minimized. 
  • BIMrx Core Advanced Parameter Manager (APM) – bug fixed for deleted unused shared parameters deleting ‘Family Type’ parameters from the project when opening the APM feature. Shared Parameters are no longer being deleted from the project when opening APM. 
  • BIMrx Core Extract & Update – fixed bug with updating values in metric. All parameter types now work with any metric value using the Extract and Update commands.  
  • BIMrx Fabrication Hanger Reporting Parameters – fixed issue with reporting the rod size parameter.  We adjusted to report the actual Geometric size of the rod as opposed to the ancillary fabrication property.  This addresses an issue when the database is updated during a project. 
    • Note: Old or outdated reporting parameters do not update after installation.  They will update when an editing event happened on the Hanger. One method is to select the hanger or multiple hangers then update one if its parameter values.  The BIMrx Reporting parameters will update as well.  
  • BIMrx Fabrication Hanger Points – fixed bug with points losing data on workshared models with multiple users accessing and synchronizing. 


BIMrx Cloud Manager Users: 


Before you install the upgrade, please take the following actions if you have customized the DismissDialogSettings.txt file.


  • When upgrading to BIMrx Revit v3.2.0.2, the settings file “dismissDialogSettings.txt” will be replaced. If you have customized the file your edits will be lost.
  • It is recommended to make a backup of C:\ProgramData\Microdesk\BIMrx.Revit\DismissDialogSettings.txt before updating to BIMrx Revit 3.2.0.2
  • Steps to save your current settings in DismissDialogSettings.txt file before installing BIMrx Revit 3.2.0.2:
    1. Copy the “DismissDialogSettings.txt” file from C:\ProgramData\Microdesk\BIMrx.Revit 
    2. Paste the “DismissDialogSettings.txt” file in C:\Users\[YOUR USER NAME]\AppData\Roaming\Microdesk\BIMrx


The v3.2.0.1 release of BIMrx® for Revit highlights the following updates:


General/Infrastructure 


  • Resolved issues with a Revit Idling Event timing out after a period of use and causing warnings in Journal Files and dialogs stating “The referenced object is not valid, possibly because it has been deleted from the database, or its creation was undone”.



BIMrx Core

  • BIMrx Core Tag All – fixed that ability to tag linked elements when any link was not loaded. Now users can tag linked elements using the Tag All command even if one of the links is unloaded.
  • BIMrx Core File Renamer – fixed the ability to rename sheets that included a period in the sheet number. Users can now rename sheets that have a period in the sheet number.
  • BIMrx Core Section – fixed the undo button text to read ‘Section Command’
  • BIMrx Core Advanced Parameter Manager – fixed the corruption of shared parameter files when trying to create a ‘Family Type’ Parameter in the shared parameter file. Users are no longer able to create a ‘Family Type’ parameter in the shared parameter file.
  • BIMrx Core Sheet View Manager – fixed conflict with have a custom parameter named ‘View Type’ in the project which caused errors in duplicating views and creating sheets. The System Parameter ‘View Type’ was renamed ‘System: View Type’ to resolve this conflict. Users will have to either re-input the view type settings in the configuration settings in the Sheet View Manager configuration settings interface or change the header of the Excel configuration file from ‘View Type’ to ‘System: View Type’ for the View Type system parameter.
  • BIMrx Core Sheet View Manager – fixed conflict when there is no titleblock family loaded in the project. This is no longer a conflict and will work even if there is no titleblock family loaded in the Revit project.
  • BIMrx Core Sheet View Manager – fixed issue with the placement module not retaining data when leaving and coming back to the module. Users can now fill in the data grid and navigate to a different module within the Sheet View Manager without losing their inputted data.



BIMrx Fabrication

  • BIMrx Fabrication Sleeve Options – Resolved issues with Sleeve Settings not saving when upgrading BIMrx from an older version. 
  • BIMrx Fabrication Hanger Array Configurations – UI update that resolves naming issue in Hanger Configurations that show Round on Rectangular Services.
  • BIMrx Fabrication Hangers – Hanger Rods reporting accurate lengths in BIMrx Parameters
