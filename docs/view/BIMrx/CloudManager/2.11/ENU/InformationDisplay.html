User Account

The User Account is the current user login name. License access to Naviate Cloud Manager is granted on a per User Account basis. 

License Type

There are two Naviate Cloud Manager license types available — Standard and Professional. They both support adding projects, files, project members and project browsing. The difference is only the Professional license provides the ability to create, deploy, execute and monitor scheduled Jobs. According to the schedule defined for each Job, they are used to upload files, download files and to publish Revit® Cloud Workshared Models. 

Please Note — The Professional license applies only to performing the above operations via a scheduled Job. The Standard license does allow interactive file uploading, downloading and model publishing from the Files (uploading) and Browse modules (downloading, model publishing).

Version

Whenever new features and/or fixes are added to Naviate Cloud Manager, a new release is issued and a new version number is assigned. This version number is based on the nature and scope of changes made to the application and consists of four digits having the following values:

    • Major: The first digit, as in 2.12.1.1, refers to the major release number. This number changes very slowly over time and indicates a highly significant change or improvement to the software— often affecting the underlying software platform and/or architecture. All users are expected to upgrade to a new major release as soon as possible once it is issued.
    • Minor: The second digit, as in 2.12.1.1, refers to the most frequent type of software change, which almost always consists of significant new features and improvements to the current major release. Each time a new minor release version is issued, the user base is notified and is expected to upgrade to the new version.
    • Release: The third digit, as in 2.12.1.1, identifies a specific release of a minor version. These often consist of fixes to reported problems and issues, but can also contain new features or improvements. When a new release is issued, users are encouraged to upgrade. Before deciding not to upgrade, however, users are strongly encouraged to view the release notes to verify there are no new features or fixes available which would benefit them.
    • Patch: (Also known as a Hotfix): The fourth digit, as in 2.12.1.1, refers to a patch or a hotfix. These are usually issued to address one or more specific problems reported by a user. Users will generally install a patch or hotfix update if they are experiencing the specific problem(s) addressed by the patch.

Build No

The build number is essentially a re-formatted statement of the Version (see preceding) and contains the same information described above.
