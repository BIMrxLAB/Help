Resolved Issues

The following issues have been resolved in this release.

    • [BBL-121] Files module: Resolved problem where project selected count total was not updating when projects were selected or unselected.
    • [BBL-122] Browse Projects module: Resolved problem in previous beta release of error caused when choosing Show Files (either Off or On) with Project Multi-Select set to On.
    • [BBL-126] After first installing the application, if an older—Version 2.1 or earlier—version of BIM360® Cloud Manager (i.e., BIM360® Bulk Loader) has been installed, when starting the app an error occurred when logged on with an Autodesk Connection and attempting to migrate the saved Custom Connection to the new format.
    • [BBL-127, BBL-128] Application crashed when doing a Select All or Unselect All from Add Projects module when no project template has been added. App now detects situation and informs the user to load a project template.
    • [BBL-129] Switching Connection Type between Autodesk and Custom for the Same Hub does not update Project Lists in the Members and Browse modules or the Account Members list.
    • [BBL-130] With both Multi-Select set to Off and Show Files set to Off (which doesn’t require or support entry of Download Type), the download process is halted because a Download Type was not entered.
    • [BBL-131] The default WinForm close button at the upper righthand corner of BIM360® Cloud Manager custom dialog boxes was redundant and has been removed. Likewise, the ‘X’ Close button—which performs the same function as the Cancel button—has also been removed to simplify use. 
    • [BBL-132] Members module: After selecting one or more projects to import members, the application was not recognizing that projects had been selected.
    • [BBL-134] Project list in Browse module was not displaying changes made affecting BIM 360 projects (projects added, deactivated, etc.) without first exiting the application or changing Hubs then changing back again.
    • [BBL-136] When Excel Project template has one or more spaces at the end of the project name it added the project to BIM 360 with the spaces still in the name.
    • [BBL-138] Application was not skipping the Plans folder in the project root when uploading from the designated local file repository.
    • [BBL-139] Files Module: Select All does not correctly performing a selection on all items.
    • [BBL-144], [BBL-166] Address Autodesk-raised Issues related to deploying BIM360® Cloud Manager to the AD App Store.
    • [BBL-146] Improve Download Job robustness and logging information.
    • [BBL-151] – Same as 132, 139
    • [BBL-152] Members: Provides a button to refresh projects and Hub and Project Members.
    • [BBL-153] Download Job: Now correctly excludes adding skipped files to the downloaded file size total to produce the correct total file size of the project.
    • [BBL-154] Handles long file names (>256 chars) in Download Jobs. Previously was skipping these files, but now renames and places them immediately below the root level folder.
    • [BBL-113] Add projects fails to add the admin user when logged on with a custom connection. This same problem also affected performing an Import and Update Members when logged on with custom connection, even though all of the preceding performed normally when logged on with an Autodesk Connection.
    • When adding projects and the system either skipped an existing project or encountered an error (i.e., due to bad or missing information on the spreadsheet template, etc.), the "Processing..." display on the current progress grid did not occur. The application now properly updates the grid to indicate "Skipped" or "Error".
    • When displaying the access level/services/roles information for each project member, the industry role list for the user now displays in alphabetical order. Some additional minor changes have also been made to the information display to enhance readability.
    • Add Projects status update display not updating correctly when projects were skipped or errors encountered.
    • [BBL-111] Addressed issue where browsing folders and files for a project was not re-enabling the project filter and clear all text button after the operation finished.
    • [BBL-103] When logged in with custom connection, the app attempted an Autodesk logon when getting the Project Role list. It now recognizes the custom connection logon and proceeds without problem.
    • [BBL-104] When adding a project with an Autodesk logon, the app prevents the user from proceeding beyond the email validation screen. It now skips the validation and adds a project either with an email address or without it (i.e., with or without an admin user). If without, the project is initially inactive.
    • [BBL-105] When selecting a project from the spreadsheet-generated grid, the user must press the check button twice for it to work. The system now works as expected when a single click on the row is made.
    • [BBL-106] The Browse Projects Show Files button was not refreshing when changed. When located on a project, toggling the Show Details button now refreshes according to the new selection while remaining on the same project.
    • [BBL-109] When downloading files from Browse Projects, the default file location was being recognized instead of the one entered by the user.
    • Resolved issue which caused app to crash when Verifying IP address or Remote Share when there was none entered for the server.
    • Added "Email From Display Name" field to email configurations settings and now uses this as the "From Name" when Job notification emails are sent, instead of erroneously displaying the smtp password.
    • When a job stops due to an error, the email body that is sent now contains the complete job statistics up to the point it stopped--number of projects processed, number of files downloaded, total skipped (see following), etc.
    • Resolved issue where files with too long of a name and path were causing the download job to fail. The system now detects when it encounters a file with too long of a Path name (248 chars max) or too long of a file name and path length (260 chars max) and skips these files. When it skips the file it includes the file name and path in the log file. Also, the download job count totals have been updated to reflect the number of skipped files.
